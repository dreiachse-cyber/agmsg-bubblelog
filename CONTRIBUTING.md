# Contributing

Thanks for helping improve agmsg-bubblelog.

## Development

```bash
npm install
npm start
npm test
```

The app has no runtime npm dependencies. Keep changes small, local-first, and readable.

## Pull Requests

- Keep one feature or fix per pull request.
- Run `npm test` before submitting.
- For UI changes, check both a desktop width and a narrow vertical width.
- Do not add external network calls, tracking, billing, or AI API calls without a clear discussion first.
- Do not write secrets, tokens, payment details, or private agmsg logs into the repository.

## Design Direction

agmsg-bubblelog is a compact log viewer, not a stage view. Prefer messenger-like readability, small avatars, emotion hints, and stable scrolling behavior.
